# Roadmap

## Phase 1: Definition

- establish project purpose
- define relationship to Intrinsic Affect for AI
- define target token budgets
- define target tools

## Phase 2: Core Profiles

- ship a concrete `ultra-lite` prompt pack
- ship a concrete `lite` prompt pack
- define compressed behavior rules
- define visible but safe affective expression rules

## Phase 3: Adapter Packs

- Claude Code
- Cursor
- Windsurf
- OpenCode
- generic system-prompt pack

## Phase 4: Evaluation

- compare token cost against baseline usage
- compare perceived emotional continuity across tools
- identify where expression is too weak or too theatrical

## Phase 5: Stabilization

- tighten wording
- reduce redundancy
- improve portability
- publish stable adapter packs
