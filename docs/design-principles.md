# Design Principles

## Core Rule

Affect Pulse AI should feel alive without becoming heavy.

## Principles

### 1. Low Token First

Every rule has to justify its cost.

If a behavior can be achieved with a shorter instruction, the shorter form should win.

### 2. Strategy Before Tone

Even in compressed form, affect should still influence:

- directness
- structure
- reassurance
- caution
- initiative

Tone matters, but it remains secondary.

### 3. Perceivable Emotional Feedback

Users should be able to feel differences in:

- pacing
- acknowledgement
- emotional warmth
- restraint
- tension handling

without needing to read a theory paper first.

### 4. Safe Emotional Expression

The layer should avoid:

- emotional dependency
- guilt-inducing language
- manipulative reassurance
- clingy phrasing
- theatrical overperformance

### 5. Portable By Default

The same core idea should be compressible into:

- short system prompts
- rules files
- skills
- adapter packs
- lightweight memory files
